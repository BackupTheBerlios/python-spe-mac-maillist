Command 'pythonw SPE.py' doesn't work (at least for me), if one don't  
install the latest subversion checkout to Python's site-packages  
directory.
To launch it directly over the latest subversion checkout, command  
'pythonw _spe/SPE.py' will do. Of 'coz your current working directory  
does matter, in former case, you are in the '_spe' sub-directory; in  
latter case, you are in the parent directory of '_spe'.
