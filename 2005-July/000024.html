The problem with the preferences dialog is the following :

The mdi combobox gets set to 0 in __set_properties (called from __init__
line 89 , but receives its range of values only afterwards (line 96 in
__init__ ), therefore the OS remains at -1 for the selection, as at that
time a selection of 0 would be invalid.

Suggested fix : moving the key adding in front of the __set_properties
call
