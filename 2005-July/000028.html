>> BTW this how I like to see error reports.

FYI, for folks on UNIXes, you might want to change your debug instructions from "python SPE.py --debug > debug.txt" to "python SPE.py --debug > debug.txt 2>&1" 

Without that '2>&1', items written to STDOUT go to the file debug.txt, and items written to STDERR (the stack trace) goes to the console, which is why you might be getting disjoint info.

Might be the same on Windows (I know '2>&1' works on windows in a command prompt, but can't test directly with SPE, as not an approved software package for use at work....)
